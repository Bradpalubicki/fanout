4. Architecture Overview

Source layout: src/app/ (routes), src/distributors/ (platform-specific posting logic), src/inngest/ (background jobs), supabase/ (migrations)

Post Pipeline: User creates post → /api/dashboard/post → stored in posts table → Inngest scheduled-post function → fan-out-post → distributor per platform → post_results recorded

Distributor Pattern: src/distributors/base.ts defines interface. Each platform (twitter.ts, linkedin.ts, etc.) implements the posting logic. 13 distributors exist (bluesky, facebook, google-business-profile, instagram, linkedin, mastodon, pinterest, reddit, threads, tiktok, twitter, youtube + base)

Inngest Functions: fan-out-post, scheduled-post, refresh-tokens, retry-failed, collect-analytics, collect-inbox, rss-auto-post, activation-nudge, trial-expiry-nudge

Multi-tenant: Clerk organizations. Each org has a profile (profiles table), OAuth tokens per platform, subscription.

Admin Panel: /api/admin/* routes for 2FA code management and OAuth app registration

Mobile API: /api/mobile/* — separate endpoint set for mobile app consumption

Public API: /api/v1/* — developer API (schedule, post, platforms, analytics)

20. Core User Journeys

  1. Sign Up → Connect Platforms: Clerk sign up → create org → /api/setup/extract-brand → OAuth connect per platform → /api/oauth/[platform]/authorize → callback → tokens stored
  2. Create & Schedule Post: Dashboard → write content → select platforms → set schedule → POST /api/dashboard/post → Inngest scheduled-post → fan-out → post_results
  3. AI Draft: Dashboard → AI Draft → POST /api/ai/draft → Claude generates platform-optimized content → approve → schedule
  4. Team Approval: Member creates post → status = pending_approval → approver reviews → /api/dashboard/approvals → approved → queued for posting
  5. Analytics: Inngest collect-analytics (11pm daily) → post engagement stored → /api/dashboard/analytics → charts in dashboard
  6. RSS Auto-Post: Add RSS feed → /api/dashboard/rss-feeds → trigger → Inngest rss-auto-post → new items auto-queued
